# What is a Database?
A **[Database](https://www.oracle.com/apac/database/what-is-database/)** is an organized collection of structured data stored electronically, designed for efficient storage, retrieval, and manipulation.
Think of it as a highly advanced, digital filing cabinet. While you might use a text document to write a letter or a simple spreadsheet to track your monthly budget, a database is designed to hold massive amounts of information, organize it logically, and allow multiple people (or computer programs) to access, search, and update that information instantly and securely.

**The Core Components:**

1. **The Data:** The actual information being stored, whether that's customer names, bank transaction records, or the inventory of a shoe store.

2. **The Database Management System (DBMS):** This is the software that acts as the "manager" of the database. It handles the complex jobs of storing the data, keeping it secure, preventing crashes, and interpreting the questions you ask it. Well-known examples include MySQL, PostgreSQL, Oracle, and MongoDB.

3. **The Applications:** The software, websites, or apps you use every day (like a banking app or an online store) that talk to the DBMS to get the information they need to show you.

# Types of Databases
Databases are organized collections of data, primarily categorized into ==relational (SQL) for structured data and non-relational (NoSQL) for unstructured data==. Common types include relational (MySQL, PostgreSQL), NoSQL (MongoDB, Cassandra), cloud-hosted (Amazon RDS), and graph databases (Neo4j), chosen based on performance, structure, and scaling needs.
##### 1. Relational Database
A relational database is ==a structured system for storing and managing data in tables with rows and columns, utilizing structured query language (SQL) for data retrieval and manipulation==.
```
Structure: Tables with rows and columns
┌──────────┬──────────┬─────────┐
│ ID       │ Name     │ Age     │
├──────────┼──────────┼─────────┤
│ 1        │ Alice    │ 25      │
│ 2        │ Bob      │ 30.     │
└──────────┴──────────┴─────────┘ 

Examples: MySQL, PostgreSQL, Oracle, SQL Server Use Case: Banking, ERP, CRM systems
```

##### 2. NoSQL Database
NoSQL databases ("not only SQL") are ==non-relational, distributed data management systems designed for high scalability, flexibility, and handling large volumes of unstructured or rapidly changing data==. Unlike SQL databases, they lack fixed, table-based schemas, enabling diverse data models like document, key-value, column-family, and graph, often prioritizing horizontal scaling and high performance over strict immediate consistency.
```
Types & Examples:
┌─────────────────┬──────────────────┬────────────────────┐
│ Type            │ Example          │ Use Case           │ ├─────────────────┼──────────────────┼────────────────────┤
│ Document        │ MongoDB          │ Content Management │
│ Key-Value       │ Redis            │ Caching, Sessions  │
│ Column-Family   │ Cassandra        │ Big Data           │
│ Graph           │ Neo4j            │ Social Networks    │ └─────────────────┴──────────────────┴────────────────────┘

MongoDB Document Example:
{
  "_id": "001",
  "name": "Alice",
  "orders": [
     {"item": "Laptop", "price": 999},
     {"item": "Mouse", "price": 29}
   ]

```

##### 3. In-Memory Database
An in-memory database (IMDB) is ==a database management system that stores data entirely in main memory (RAM) rather than on disk==, providing ultra-fast, low-latency data retrieval. By eliminating disk I/O, they power real-time analytics, gaming, IoT, and high-speed transactions. Key options include Redis, SAP HANA, and [Oracle Database In-Memory](https://www.oracle.com/database/in-memory/).
```
Data stored in RAM (not disk)
┌─────────────────────────────────────┐
│ Disk DB: Read → 100ms               │
│ In-Memory: Read → 0.1ms ⚡           │
└─────────────────────────────────────┘
Examples: Redis, Memcached, VoltDB
Use Case: Real-time analytics, Gaming leaderboards
```

##### 4. Cloud Database
A cloud database is ==a database designed, deployed, and accessed through a cloud environment==. Cloud databases organize and store structured, unstructured, and semi-structured data just like traditional on-premises databases. However, they also provide many of the same benefits of [cloud computing](https://cloud.google.com/learn/what-is-cloud-computing), including speed, scalability, agility, and reduced costs.

```
│ Azure        │ Azure SQL, Cosmos DB      │ └──────────────┴───────────────────────────┘
Benefits: Scalable, Pay-as-you-go, Managed
```


# DBMS vs RDBMS

```
┌──────────────────────────────────────────────────────┐
│                         COMPARISON                   │
├─────────────────┬────────────────┬───────────────────┤
│ Feature         │ DBMS           │ RDBMS             │ ├─────────────────┼────────────────┼───────────────────┤
│ Data Storage    │ Files/Hierarch │ Tables (Relations)│
│ Relationships   │ Not enforced   │ Enforced via Keys │
│ Data Redundancy │ High           │ Low (Normalized)  │
│ SQL Support     │ No/Limited     │ Yes (Full SQL)    │
│ ACID            │ Not guaranteed │ Fully Supported   │
│ Normalization   │ Not Supported  │ Supported         │
│ Security        │ Low            │ High              │
│ Examples        │ XML, File Sys  │ MySQL, PostgreSQL │ └─────────────────┴────────────────┴───────────────────┘
```

##### DBMS (Database Management System)
A ==Database Management System== (DBMS) is software designed to efficiently create, store, manage, and retrieve data, serving as an interface between users and the database. It ensures data security, integrity, and consistency while reducing redundancy across applications. Popular systems include MySQL, PostgreSQL, Oracle, and MongoDB.
*Software that manages databases **without** enforcing relationships between data.*
##### RDBMS (Relational Database Management System)
A ==Relational Database Management System== (RDBMS) is a software system that manages data structured in tables with rows and columns, allowing for efficient storage, retrieval, and updates. It uses Structured Query Language (SQL) to manage data, ensuring data integrity and relationships between tables using keys. Common examples include MySQL, PostgreSQL, Oracle Database, and Microsoft SQL Server.
*Software that manages databases **with** strict relationships, using tables and SQL.*

```
-- RDBMS Example: Enforcing Relationships
CREATE TABLE customers (
  customer_id INT PRIMARY KEY, -- Enforced uniqueness
  name VARCHAR(100) NOT NULL
);

CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  customer_id INT,
  FOREIGN KEY (customer_id) -- Enforced relationship
      REFERENCES customers(customer_id)
);
```


# RDBMS Core Concepts
RDBMS (Relational Database Management System) core concepts center on ==organizing data into structured, related tables (rows and columns) to ensure data integrity and efficient retrieval==. Key elements include tables (relations), primary/foreign keys, ACID properties for reliable transactions, normalization to eliminate redundancy, and SQL (Structured Query Language) for management.

###### Tables, Rows, Columns, Schema, Instance
```
DATABASE: company_db
│
├── SCHEMA: hr_schema
│ └── TABLE: employees
│ ┌──────────┬──────────┬────────────┬──────────┐
│ │ emp_id   │ name     │ department │ salary.  │ ← COLUMNS
│ │ (INT)    │(VARCHAR) │ (VARCHAR)  │(DECIMAL) │ ← DATA TYPES
│ ├──────────┼──────────┼────────────┼──────────┤
│ │ 1        │ Alice    │ IT         │ 75000    │ ← ROW/RECORD
│ │ 2        │ Bob      │ HR         │ 65000    │ ← ROW/RECORD
│ │ 3        │ Carol    │ IT         │ 80000    │ ← ROW/RECORD
│ └──────────┴──────────┴────────────┴──────────┘
│       ↑ 
     INSTANCE = Current data at this moment
```

###### Syntax:
```
-- Creating Schema and Table
CREATE DATABASE company_db;
USE company_db;

CREATE TABLE employees ( -- TABLE
  emp_id INT, -- COLUMN 1
  name VARCHAR(100), -- COLUMN 2
  department VARCHAR(50), -- COLUMN 3
  salary DECIMAL(10,2) -- COLUMN 4
);

-- Inserting ROWS
INSERT INTO employees VALUES (1, 'Alice', 'IT', 75000);
INSERT INTO employees VALUES (2, 'Bob', 'HR', 65000);
INSERT INTO employees VALUES (3, 'Carol', 'IT', 80000);
```

```

```

# Data vs. Information

| Aspect              | Data                   | Information                |
| ------------------- | ---------------------- | -------------------------- |
| **Definition**      | Raw, unprocessed facts | Processed, meaningful data |
| **Form**            | Unorganized            | Organized                  |
| **Meaning**         | No context             | Has context                |
| **Example**         | `90, 45, 78, 23`       | `Average score = 59`       |
| **Dependency**      | Independent            | Depends on Data            |
| **Decision Making** | Cannot be used         | Can be used                |

**Example:**
```
┌─────────────────────────────────────────────┐
│ DATA → PROCESS → INFORMATION                │
│ "23, Male, → Organize → "John is a          │
│ John" → Analyze → 23yr old male".           │ └─────────────────────────────────────────────┘
```
