# What is a Database?
A **Database** is an organized collection of structured data stored electronically, designed for efficient storage, retrieval, and manipulation.

# Data vs. Information

|Aspect|Data|Information|
|---|---|---|
|**Definition**|Raw, unprocessed facts|Processed, meaningful data|
|**Form**|Unorganized|Organized|
|**Meaning**|No context|Has context|
|**Example**|`90, 45, 78, 23`|`Average score = 59`|
|**Dependency**|Independent|Depends on Data|
|**Decision Making**|Cannot be used|Can be used|

```
Example:

│ DATA → PROCESS → INFORMATION       │

```


# DBMS vs RDBMS

##### DBMS (Database Management System)
A Database Management System (DBMS) is ==software designed to efficiently create, store, manage, and retrieve data, serving as an interface between users and the database==. It ensures data security, integrity, and consistency while reducing redundancy across applications. Popular systems include MySQL, PostgreSQL, Oracle, and MongoDB.
*Software that manages databases **without** enforcing relationships between data.*
##### RDBMS (Relational Database Management System)
A ==Relational Database Management System== (RDBMS) is a software system that manages data structured in tables with rows and columns, allowing for efficient storage, retrieval, and updates. It uses Structured Query Language (SQL) to manage data, ensuring data integrity and relationships between tables using keys. Common examples include MySQL, PostgreSQL, Oracle Database, and Microsoft SQL Server.
*Software that manages databases **with** strict relationships, using tables and SQL.*

```
-- RDBMS Example: Enforcing Relationships
CREATE TABLE customers (
  customer_id INT PRIMARY KEY, -- Enforced uniqueness
  name VARCHAR(100) NOT NULL
);

CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  customer_id INT,
  FOREIGN KEY (customer_id) -- Enforced relationship
      REFERENCES customers(customer_id)
);
```
